Project Description

Vantage is a cloud-based monitoring and change detection platform that will be deployed on a Linux virtual machine and made accessible through a public IP address and domain name. The purpose of the project is to build a practical system that monitors both internal server activity and selected external web content from a single hosted environment. The platform will collect and display useful system information such as CPU usage, memory usage, disk usage, service availability, and authentication log activity. These checks will be automated using scripts that run on the server at regular intervals, reducing the need for manual monitoring and demonstrating practical server administration skills. In addition to internal system monitoring, Vantage will periodically check selected public web pages and record when important content changes occur. This will allow the project to demonstrate not only Linux server deployment and web hosting, but also automation, logging, and change detection in a real cloud environment. A realistic next stage of development will be to present the collected results through a simple web dashboard, improve the structure of stored monitoring logs, and strengthen the server by configuring HTTPS and basic security hardening measures. In this way, the project provides a clear pathway from basic cloud deployment to a more advanced monitoring platform that reflects both technical development and practical cybersecurity application.

In its initial stage, the project will include a working Linux virtual machine, a configured Nginx web server, DNS linkage to a domain name, and a public webpage introducing the system. This first phase focuses on the core cloud deployment requirements of the unit, including provisioning the server, connecting remotely through SSH, installing and configuring software manually, and making the service accessible over the internet. A scripting component will then be added to automate practical monitoring tasks such as checking whether key services are running, collecting system resource usage, and reading selected authentication log activity.

As development continues, Vantage will be expanded into a more structured monitoring platform with a stronger web-based presentation of collected results. The next stage will focus on storing monitoring outputs in an organised way, presenting them through a simple dashboard, and adding external web page change detection so that selected public pages can be checked for meaningful updates over time. Further improvement will include HTTPS configuration, stronger server hardening, and cleaner reporting of monitoring events. This development path keeps the project technically focused, realistic to implement, and closely aligned with Linux administration, automation, and cybersecurity practice.

Why DigitalOcean Was Chosen

DigitalOcean was selected as the cloud provider because it offers a clear and practical Infrastructure as a Service environment with full SSH access, which matches the technical requirements of this project. This allows the server to be configured manually rather than relying on a pre-built managed platform, making it well suited for demonstrating Linux administration, software installation, web hosting, and cloud deployment in a hands-on way. That level of control is valuable because the aim of the project is not only to host a webpage, but also to build and manage the underlying server environment directly.

A further reason for choosing DigitalOcean is that its interface is well organised, its documentation is clear, and a virtual machine can be deployed with relatively little overhead compared with more complex cloud platforms. This makes it a suitable option for a student project, as it allows more attention to be given to the monitoring and automation features rather than to the setup process itself. In addition, DigitalOcean works well with the GitHub Student Developer Pack, which helps reduce costs and makes it realistic to continue developing the project after the initial proposal stage.

Planned Development Stages

  • Stage 1: Deploy the Ubuntu virtual server, configure Nginx, publish the proposal webpage, connect the server through SSH, and document the setup process on GitHub.
  • Stage 2: Add scripts to monitor key system information such as service availability, CPU usage, memory usage, disk usage, and selected authentication log activity, then organise the collected results clearly.
  • Stage 3: Extend the platform by adding external web page change detection, a simple dashboard-style presentation of monitoring results, HTTPS configuration, and stronger server hardening measures.
